Git Comandos uwu

- Git init
Crea el staging y el repositorio en la ram para posteriormente ser utilizado con estos codigos,Esto es lo primero que se debe crear.

- Git add .
Manda el archivo al staging para ser rastreado y despues enviado al repositorio.

- Git commit -m "Nombre de la version del archivo"
Envia el archivo al repositorio comenzando el "historial de cambios".

- Git commit -am "Nombre de la version del archivo"
Envia el archivo al repositorio sin necesidad de enviarlo al staging (solo se puede hacer si ya has guardado versiones anteriores con el "git add .").

- Git show
Muestra los ultimos cambios hechos en el archivo.

Git log / Git log --oneline / Git log --stat
Muestra el historial de cambios o los commits que has hecho / hace lo mismo pero se ve mas bonito / hace lo mismo pero siendo mas detallado.

Git diff
Muestra los cambios entre el archivo enviado al staging y el archivo actual del disco duro sin necesidad de enviarlo al commit (esto no implica que no se deba enviar los archivos al repositorio, esto es importante debido al historial).

Git diff "Codigo de la version" "nombre del archivo"
Muestra los cambios entre dos archivos de diferente orden cronologico enviados al repositorio.

Git checkout "Codigo de la version" "nombre del archivo"
Se viaja en el tiempo :v, volviendo al archivo que eligamos, si se quiere guardar se hace con el "git command -am"

Git branch "nombre de la rama"
Crea una rama.

Git branch
Muestra las ramas creadas.

Git merge "nombre de la cabecera"
Une dos ramas creadas con anterioridad.

Git pull origin master / Git pull origin master --allow-unrelated-histories
Trae el repositorio de github al repositorio local.

Git push origin master
Envia el repositotio local al repositorio de github.
